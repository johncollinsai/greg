AI analyst for due diligence

Welcome to the AI analyst for due diligence demo. In 1-2 years (if not much earlier), I expect everyone will run personalized AI assistants and image/video/music generator models either directly, or indirectly via other applications and tools, in their browser and smartphone. Imho, this will lead to profound change in work and personal behaviors: you won't need to use search, nor workflow queues; and in your personal life you won't need to book flights or restaurants, nor go to shopping sites yourself. Personalized AI assistants will do this for you.

There is always risk associated with new technology adoption and AI is no different. It is necessary to embrace AI in order to understand the risks and how to mitigate them. Sitting back and waiting simply means missing the boat. Fine-tuning an open-source LLM, such as Meta's LLaMA or Stanford's Alpaca, is a great way to tailor these models to suit specific business requirements (credit risk, collateral management, to name just two) and mitigate certain risks, so I'm going to fine-tune a 13B LLaMA and apply it to my favorite problem, volatility (i.e., risk) prediction, in order to take a deeper look.

In this demo I use various approaches to leveraging pretrained LLMs for novel tasks, including in-context learning, finetuning, parameter-efficient finetuning, quantization, and reinforcement learning with human feedback. These approaches offer different ways to optimize model performance while minimizing computational costs.

User guide

How to use your AI analyst for due diligence

This AI assistant is a composite of several AI applications and a variety of HTML and JavaScript-based React components. Each of the AI applications is architected as a microservice, functioning independently while also forming part of an extensible integrated ecosystem.

  • The front end uses React for routing, state management, AJAX requests, and UI to provide intuitive and responsive UI. A Flask app serves as an API to the React frontend.
  • Best practices are strictly followed for both client-side (React) and server-side (Flask) security. JSON Web Tokens (JWT) provide robust API authentication. Rate limiting and input validation protect not only against SQL injection, cross-site scripting, and other similar vulnerabilities, but also provide rigorous prompt boundaries, a cornerstone of responsible AI.
  • The AI assistant is currently hosted on Google Cloud, utilizing Nginx as a web server and proxy, and orchestrated through Kubernetes (K8s) for managing the microservices, ensuring smooth deployment, scaling, and networking.
  • The source code that drives this demo resides across multiple repositories on my GitHub. This multi-repo strategy facilitates modular development and efficient version control. There are no black boxes: If you're interested in delving deeper into the mechanics behind the services you see here, you're welcome to explore the GitHub repositories to understand the codebase further.
